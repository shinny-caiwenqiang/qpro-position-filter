# QPRO 持仓字段筛选

QPRO 数据只保留在用户电脑上。本仓库的 GitHub Pages 是下载与说明页；实际查询必须由本地筛选服务执行。

## 用户使用流程

1. 在 QPRO 中启用本地 API，确认端口为 `8811`。
2. 在 GitHub 页面下载 Windows 启动器并解压。
3. 双击 `qpro-position-filter.exe`；无需安装 Node.js。
4. 启动器自动打开 `http://127.0.0.1:8812/`，再进行持仓筛选。

## 为什么不直接在 GitHub Pages 查询

浏览器会限制 HTTPS 网页访问本机 `127.0.0.1`，且 QPRO 数据 API 必须保持本地私有。GitHub Pages 因此只作为分发入口，不直接读取交易数据。

## 安全边界

- 本地服务仅监听 `127.0.0.1`。
- QPRO 的 `8811` API 不对公网暴露。
- 页面不上传账号、令牌或持仓数据。
